# Phase 6.7 — Modulation Ring UI Activation Implementation Plan

> **STATUS:** IMPLEMENTED. Authoritative status: `docs/superpowers/STATUS.md`.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Activate the Modulation Ring (S/H, Sync 1/16, Legato) for the parameter categories
that Phase 1 already scaffolded (curve-node Y values). Phase 1 ships the visual toggles in
a non-functional state; Phase 6.7 makes them do something audible by reading `ctx.bpm`,
`ctx.beat_position`, and `ctx.midi_held_notes` from `ModuleContext`.

**Architecture:** The ring's per-(slot, curve, node) state moves out of the editor's
`HashMap<…, ModRingState>` into a persisted, audio-thread-readable `RingStateBank` snapshot
that the Pipeline reads each block. The Pipeline then *transforms* the curve-Y values it
copies into `slot_curve_cache` according to each ring entry's flags: S/H latches values
at the next 16th-note tick, Legato ramps between latches, Sync sets the tick rate.

The transformation lives in `dsp/modulation_ring.rs`. Per-block state (last latched value,
last tick beat-position) lives on the Pipeline so it survives across blocks.

**Tech Stack:** Rust, the `ModRingState` struct + `mod_ring_states` editor map from
Phase 1, the `ctx.bpm` / `ctx.beat_position` populated by Phase 5b BPM sync, the
`ctx.midi_held_notes` populated by Phase 6.3.

**Depends on:** Phase 1 (ring UI scaffolding + `ModRingToggle` enum + per-(slot,curve,node)
state map), Phase 5b BPM sync (so `ctx.bpm` and `ctx.beat_position` are populated by the
host transport), Phase 6.3 (so `ctx.midi_held_notes` is populated for note-driven S/H).

## Out of scope

- **Sensitivity (cumulative-push) flag** — defer; not in Phase 1 scaffold.
- **Wiring the ring to other parameter categories** (curve transforms, matrix sends, module
  params) — v1 ships only curve-node Y values. The remaining ~29 categories land
  incrementally in follow-up patches.
- **Per-toggle context menu** (Sync 1/16 → 1/8 / 1/4 / 1/32 etc.) — defer; v1 hardcodes 1/16.
- **Random-source S/H** — v1 sample-and-holds the *user's current value*, not a random
  modulator output. Random-source S/H requires a separate "random LFO" infra and is out
  of scope for this plan.
- **Polyphonic per-note ring state** — single global ring state per parameter; per-voice
  state would need MIDI-channel demuxing.

//! Modulation Ring (S/H, Sync 1/16, Legato) — audio-thread-readable bank.
//!
//! Phase 1 shipped the visual toggles in `editor::mod_ring`. This module moves
//! the state from the editor's local map into a persisted, audio-thread-readable
//! struct so the Pipeline can apply the modulators each block.

use crate::editor::{ModRingState, ModRingToggle};
use std::collections::HashMap;

#[derive(Clone, Copy, Debug, Hash, PartialEq, Eq)]
pub struct RingKey {
    pub slot:  u8,
    pub curve: u8,
    pub node:  u8,
}

#[derive(Default, Clone)]
pub struct RingStateBank {
    entries: HashMap<RingKey, ModRingState>,
}

impl RingStateBank {
    pub fn entry_count(&self) -> usize { self.entries.len() }

    pub fn get(&self, key: RingKey) -> ModRingState {
        self.entries.get(&key).copied().unwrap_or_default()
    }

    pub fn set_toggle(&mut self, key: RingKey, t: ModRingToggle, on: bool) {
        let mut s = self.get(key);
        if on { s.set(t); } else { s.clear(t); }
        if s.is_empty() {
            self.entries.remove(&key);
        } else {
            self.entries.insert(key, s);
        }
    }

    /// Iterate all entries — used by the Pipeline per-block snap.
    pub fn iter(&self) -> impl Iterator<Item = (&RingKey, &ModRingState)> {
        self.entries.iter()
    }

    /// Clear all entries (for reset-to-default button).
    pub fn clear(&mut self) { self.entries.clear(); }
}
```

#[derive(Clone, Copy, Debug)]
pub struct RingTransformState {
    /// Last value latched by S/H. Sentinel value f32::NAN means "not yet latched".
    latched_value:    f32,
    /// Beat position at last S/H latch event. -1.0 means "never latched".
    last_latch_beat:  f32,
    /// Previous block's *output* value — used by Legato to compute per-sample ramps.
    prev_out_value:   f32,
}

impl Default for RingTransformState {
    fn default() -> Self {
        Self {
            latched_value:    f32::NAN,
            last_latch_beat:  -1.0,
            prev_out_value:   f32::NAN,
        }
    }
}

impl RingTransformState {
    pub fn is_latched(&self) -> bool { !self.latched_value.is_nan() }
    pub fn latched_value(&self) -> f32 {
        if self.latched_value.is_nan() { 0.0 } else { self.latched_value }
    }
    pub fn last_latch_beat(&self) -> f32 { self.last_latch_beat }
    pub fn set_latched(&mut self, value: f32, beat: f32) {
        self.latched_value = value;
        self.last_latch_beat = beat;
    }
    pub fn set_prev_out(&mut self, value: f32) { self.prev_out_value = value; }
    pub fn prev_out(&self) -> f32 {
        if self.prev_out_value.is_nan() { 0.0 } else { self.prev_out_value }
    }
}
```

/// Returns true if a tick boundary at `period` beats was crossed between
/// `last_beat` (inclusive) and `current_beat` (exclusive).
///
/// Conventions:
/// - `last_beat < 0.0` is the sentinel "never latched"; always returns true so
///   the first block latches.
/// - If `current_beat < last_beat`, the host transport looped — treat as a cross.
#[inline]
pub fn crossed_tick_at_beat(last_beat: f32, current_beat: f32, period: f32) -> bool {
    if last_beat < 0.0          { return true; }
    if current_beat < last_beat { return true; } // loop wrap
    let last_tick    = (last_beat    / period).floor();
    let current_tick = (current_beat / period).floor();
    current_tick > last_tick
}
```

pub struct RingApplyArgs {
    pub ring:          ModRingState,
    pub input_value:   f32,
    pub current_beat:  f32,
    pub block_samples: usize,
}

/// Apply ring modulators to a single parameter for one audio block.
///
/// This is the small kernel called once per (slot, curve, node) per block when
/// the Pipeline finds a non-empty ring state. It mutates `state` in place.
///
/// For ergonomics, returns a fixed `[f32; 4096]` (the maximum block size we
/// support). The caller copies into its destination using `block_samples`.
pub fn apply_ring(state: &mut RingTransformState, args: RingApplyArgs) -> [f32; 4096] {
    debug_assert!(args.block_samples <= 4096);
    let mut out = [0.0_f32; 4096];

    let sh   = args.ring.is_set(ModRingToggle::SampleHold);
    let sync = args.ring.is_set(ModRingToggle::Sync);

    let period = if sync { 0.25 } else { 1.0 };

    // Decide whether to latch *now*.
    let should_latch = if !sh {
        true // pass-through: always re-latch the input.
    } else {
        crossed_tick_at_beat(state.last_latch_beat, args.current_beat, period)
    };

    if should_latch {
        state.set_latched(args.input_value, args.current_beat);
    }

    let target = state.latched_value();

    // Legato is wired in Task 5 — for now, fill with the held value.
    for i in 0..args.block_samples { out[i] = target; }

    state.set_prev_out(target);
    out
}
```

## Task 6: Pipeline integration — apply ring to curve-node Y values

**Files:**
- Modify: `src/dsp/pipeline.rs` (per-block ring snap + transform pass)
- Modify: `src/editor/curve.rs` (expose per-node Y access for ring transform)

The Pipeline already builds `slot_curve_cache` from triple-buffered curves. After that
build, iterate ring entries and patch the curve cache: replace the per-bin contribution
of node `node` for slot `slot` curve `curve` with the ring-transformed Y.

For v1, the simplest effect path is: clamp the *final per-bin curve cache value* by the
ring-transformed scalar. Per-block the ring writes one scalar (the held node Y); we then
multiply the curve cache for that slot/curve uniformly by `ring_value / live_value` so
the curve evaluator's relative shape is preserved but its midpoint follows the ring.

This is a one-line patch that works without exposing per-node internals.

```rust
// Modulation Ring pass — applies S/H, Sync, Legato to live node Y values.
let ring_snapshot = self.shared.ring_states
    .try_lock()
    .map(|g| g.clone())
    .unwrap_or_default();

let bpm = ctx_bpm;            // already snapped earlier in process()
let beat = ctx_beat_position; // ditto
let block_samples = self.last_block_size;

for (key, ring_state) in ring_snapshot.iter() {
    if ring_state.is_empty() { continue; }
    let key = *key;
    let live_y = self.read_live_node_y(key);
    let state = self.ring_transforms.entry(key).or_default();
    let out = crate::dsp::modulation_ring::apply_ring(
        state,
        crate::dsp::modulation_ring::RingApplyArgs {
            ring: *ring_state,
            input_value: live_y,
            current_beat: beat,
            block_samples,
        },
    );
    let scaled_y = out[block_samples.saturating_sub(1)];

    // Scale the existing curve cache for this slot/curve uniformly so the
    // curve evaluator's relative shape is preserved but the midpoint follows
    // the ring's held value. live_y == 0.0 ⇒ skip to avoid div-by-zero.
    let scale = if live_y.abs() < 1e-9 { 1.0 } else { scaled_y / live_y };
    if (scale - 1.0).abs() < 1e-6 { continue; }
    let s = key.slot as usize; let c = key.curve as usize;
    if s >= self.slot_curve_cache.len() { continue; }
    if c >= self.slot_curve_cache[s].len() { continue; }
    let cache = &mut self.slot_curve_cache[s][c];
    for v in cache.iter_mut() { *v *= scale; }
    let _ = bpm;
}

// Drop ring_transforms entries whose ring state has become empty.
self.ring_transforms.retain(|k, _| {
    ring_snapshot.get(*k).is_set(ModRingToggle::SampleHold)
        || ring_snapshot.get(*k).is_set(ModRingToggle::Sync)
        || ring_snapshot.get(*k).is_set(ModRingToggle::Legato)
});
```

    self.params.curve_node_y(s, c, n)
}
